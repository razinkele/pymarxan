# Real-world Marxan format fixtures (synthetic)

These two fixtures are **synthetic** — small, deterministically generated
projects, not third-party data — but they reproduce the file-format
*variations* found in real public Marxan datasets so the loader is exercised
end-to-end (load → solve) against them. They carry **no third-party data and no
license encumbrance**.

Generated by [`_generate.py`](_generate.py) (run from the repo root:
`python tests/data/formats/_generate.py`); exercised by
[`tests/test_e2e_marxan_formats.py`](../../test_e2e_marxan_formats.py).

Both encode the same project: a 5×5 grid of 25 planning units, 4 features
(`prop`-based targets, high SPF), and 4-neighbour boundaries.

| Fixture | Format quirk reproduced | Seen in the wild in |
|---|---|---|
| `double_tab/` | columns separated by repeated tabs (`id⇥⇥cost`); boundary column named `bound` | MarOpt / AdrienBrunel `reserve-site-selection` |
| `quoted_csv/` | quoted `puvspr` header (`"species","pu","amount"`) with comma rows; boundary column named `boundary` | MarxanConnect tutorials |

The comma/plain-tab base format is already covered by `tests/data/simple`
(prioritizr-style). The format-parsing fixes these guard against landed in the
v0.8.3 loader-robustness work (whitespace-run collapsing + `bound`/`boundary`
alias).

We deliberately do **not** bundle the actual public datasets: prioritizr is
GPL-3 and AdrienBrunel's `reserve-site-selection` carries no license, both
incompatible with vendoring into this MIT-licensed repository. MarxanConnect is
MIT, but a synthetic fixture covers its format quirk without adding a data
dependency.
